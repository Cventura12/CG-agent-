# GC Agent Positioning Memo

## Core Positioning

GC Agent should be framed as software that helps a general contractor keep control of messy project communication and financial follow-through once work is underway.

It should not be framed as:

- a generic AI agent for contractors
- a broad construction copilot
- a preconstruction document-review product
- an autonomous estimator

The sharper description is:

> GC Agent captures messy project communication, detects unresolved changes and commitments, and turns them into tracked actions, documentation, quote drafts, follow-up, and job history.

## Product Category

Execution-side communication control for general contractors.

## The Problem We Solve

Important project and financial information is scattered across:

- phone calls
- texts
- field notes
- uploads
- superintendent updates
- owner requests
- follow-up threads

The result is expensive:

- scope changes get missed
- unresolved items slip
- quotes stall
- follow-up dies
- office and field lose alignment
- financially important communication is never turned into action

## Product Promise

GC Agent helps a contractor:

1. capture what happened
2. detect what changed
3. surface what is unresolved
4. create the next operational or financial step
5. preserve follow-through and accountability

## What Stays Core

- transcript ingest
- queue and review workflow
- quote draft generation, review, send, and follow-up
- transcript-to-quote prefill
- job detail timeline and communication history
- spreadsheet pricing import
- delivery and follow-up tracking
- auditability and traceability

## What Gets Reframed

### Quotes

Quotes stay important, but they are not the whole product identity.

The right framing is:

- turn messy communication into a reviewable quote draft

not:

- autonomous AI estimating

### Memory

Memory is useful, but it should be explained as:

- contractor pricing baseline
- learned patterns from real outcomes
- evidence for quote confidence

not:

- abstract agent intelligence

### Analytics

Analytics should help the office make decisions, not just show decorative metrics.

The strongest analytics themes are:

- quote turnaround
- queue backlog
- follow-up effectiveness
- transcript linkage/actionability
- unresolved operational pressure

## What Gets Deprioritized

- generic AI-agent messaging
- broad preconstruction intelligence framing
- document-review-first positioning
- live voice-agent ambition before transcript/action loops are trusted
- decorative insights that do not drive action

## Buyer-Level Message

The most honest buyer-facing statement is:

> GC Agent helps your team keep control of calls, texts, field notes, uploads, and updates by turning them into tracked actions, quote work, follow-up, and job history.

## Internal Decision Filter

When evaluating new features, ask:

1. Does this help capture messy communication?
2. Does this surface unresolved items or changes?
3. Does this improve follow-through or financial control?
4. Does this strengthen the job record or queue?
5. Does this make office/field coordination more reliable?

If the answer is no, the feature is probably not core to current positioning.

## One-Line Product Summary

GC Agent turns project communication into tracked operational action.
