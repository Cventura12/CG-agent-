# Arbor Sprint Board: Now / Next / Later

Last updated: 2026-03-21

## Now

These are the highest-leverage tasks because they directly improve trust in the core product loop.

### 1. Make the field update -> queue -> draft -> approval -> job-history loop airtight

Why now:

- this is the flagship workflow
- it is the clearest product value
- if this is weak, the rest of the system feels ornamental

Success looks like:

- inbound update appears in queue
- approval creates the right next artifact
- draft output ties back to the job
- job history shows what happened
- no ambiguous state remains behind

### 2. Make queue states explicit and trustworthy

Why now:

- the queue is where trust is earned or lost

Success looks like:

- pending vs approved vs dismissed vs snoozed is always clear
- low-confidence items are visible
- operators know what requires manual review

### 3. Improve operator trust in voice outcomes

Why now:

- voice is differentiated, but still needs more operational confidence

Success looks like:

- every live call leaves a useful operational artifact
- low-confidence voice sessions degrade safely
- operators can immediately see what the call produced

## Next

These come right after the core loop is tightened.

### 4. Make Jobs the operating truth center

Success looks like:

- every job shows what changed recently
- unresolved items are visible
- blocked next steps are visible
- approved actions leave a clean trail

### 5. Improve draft quality

Success looks like:

- quote drafts feel useful, not generic
- follow-up drafts feel tied to the job context
- scope extraction is tighter and more specific

### 6. Add clearer ROI and proof surfaces

Success looks like:

- the owner can see what Arbor caught
- protected revenue is visible
- follow-up slippage prevented is visible

## Later

These matter, but they should not distract from the main loop first.

### 7. Expand analytics depth

Only after the workflow is trustworthy.

### 8. Broaden document intelligence

Only after the main quote/review path is strong.

### 9. Add broader collaboration and permissioning depth

Only after single-office/operator trust is established.

### 10. Push voice toward full best-in-class conversational quality

This stays important, but it should build on a dependable operational foundation rather than outrun it.

## Rule

If a task does not make Arbor:

- more trustworthy
- more operationally useful
- more likely to prevent missed billing or follow-through

it should not outrank the current Now list.
